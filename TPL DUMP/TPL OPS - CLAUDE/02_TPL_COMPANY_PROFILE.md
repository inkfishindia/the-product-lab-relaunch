## OPERATIONAL MODEL

### Core Philosophy
- **Print-on-Demand / Just-in-Time**: Almost all production is made-to-order
- **Rush-First Reality**: "Almost everything is a rush order" (Dan's words)
- **Creative-First Business**: Design quality drives business value
- **Batch Production**: Factory routing is product-based, not client-based — all keychains go to the keychain factory regardless of which client ordered them

### Seasonal Patterns
- Seasonal lift during Indian festivals
- Demand fluctuations requiring capacity flexibility

### Sales Channel Sources
- B2B Portal
- Website
- Direct / Store
- Artist Collaborations

## KEY OPERATIONAL CHALLENGES (DOCUMENTED)

| Challenge | Impact | Status |
|-----------|--------|--------|
| 60% of orders delayed beyond promised dates | Client satisfaction, revenue | Being addressed via bin system |
| Communication gaps between team members | Workflow bottlenecks | Automation + bin system planned |
| "Goldfish memory" team | Repeated knowledge loss | Bin system = persistent visual cues |
| 15% quality rejection rate | Cost, rework, client trust | QC gates being implemented |
| No automated notifications | Manual coordination overhead | Smart triggers designed |
| No individual product profitability tracking | Margin blindness | Database fields designed |
| 2 design revision cycles (B2B and B2C) | Timeline pressure | Process being standardized |
| Data/knowledge transfer failures | Operational fragility | Persistent systems being built |
