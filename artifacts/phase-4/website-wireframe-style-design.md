## Direction summary

The website should follow the hybrid direction already implied by strategy and creative:

**Darkroom atmosphere + Type Pressure hierarchy**

That means:

- dark visual field
- products lit like they matter
- typography treated as a major visual object
- red and yellow used as signal colors, not decoration
- mobile-first page rhythm with strong editorial pacing

The coming soon page is the first live proof of that system.

---

## Visual principles

### 1. Big type before busy layouts

The site should feel immediate. Headlines do the work first. Product grids and metadata come second.

### 2. Objects on a stage

Products are small. The interface should give them contrast and gravity instead of floating them in bland whitespace.

### 3. Sharp, not cute

Default geometry is hard-edged. Buttons and inputs can have minimal radius for usability, but the overall system should stay angular.

### 4. Copy leads the frame

Collection names and page intros should name the person or attitude first. Product-type language is secondary.

### 5. Every page must help the ₹499 threshold

The UI should repeatedly surface:

- bundles
- paired-product suggestions
- gifting routes
- free-shipping logic

## Homepage wireframe

### Section order

1. Sticky nav
2. Hero statement
3. Card sticker hero collection
4. No Filter collection
5. Opinion bar
6. Trust block
7. Gifting route
8. Drop teaser
9. Footer

### Desktop layout

- Hero split: large headline on left, product-led visual or teaser system on right
- Featured collections in 4-column grids
- Opinion bar should interrupt the scroll with a full-width statement

### Mobile layout

- Hero stacks vertically
- Collection grids collapse to 2 columns
- CTA buttons stay full width
- Sticky bottom nav supports Shop / Drop / Gifts / Search

## Collection page wireframe

### Section order

1. Collection statement
2. Short intro about who it is for
3. Product grid
4. Bundle nudge
5. Related collection chips

### Layout behavior

- The first screen should not begin with filtering controls
- It should begin with an attitude statement
- Filters come after the visitor understands the collection point of view

### Design note

Collection pages should feel like a sharper continuation of the coming-soon selector states. Each collection is effectively a scaled-up lane.

---

## Product page wireframe

### Section order

1. Massive product title
2. Product image block
3. Identity hook
4. Price and add-to-cart
5. WhatsApp-shareable line
6. Specs
7. Artist attribution
8. Related products / bundle suggestion

### Layout behavior

- On desktop, title and visual can sit in tension side-by-side
- On mobile, title should still feel oversized and poster-like
- The add-to-cart zone must stay obvious and close to the price

### Design note

The title should not feel like a label beneath a photo. It should feel like part of the art direction.

## UI style notes

### Type

- Headings: Barlow Condensed, heavy, uppercase
- Body: Inter
- Metadata / labels: Space Mono

### Color behavior

- Black is the field
- Warm white is the main text color
- Red is for action and heat
- Yellow is for emphasis and timing

### Motion

- Strong page-load reveal
- State transitions on hover/tap
- No playful bouncing
- Movement should feel deliberate and slightly severe

### Components

- Buttons should feel compact and forceful
- Cards should read as panels, not soft containers
- Tags should be mono and utility-like

---

## Relationship to the coming-soon page

The coming-soon page should preview the website system in miniature:

- same typography
- same color logic
- same sharp geometry
- same identity-first copy strategy

If someone lands on the coming-soon page and later reaches the live site, it should feel like the same brand opening into a larger world, not like a campaign landing page attached to a different store.
