# Phase 2: Storefront MVP

<!-- last-updated: 2026-05-29 -->

## Objective

Build the first launch storefront on Next.js using Medusa as the commerce backend.

## Pages

| Page | Required? | Notes |
|---|---:|---|
| Homepage | Yes | Opinion-led entry point |
| Collections | Yes | Browse by attitude/persona first |
| Collection detail | Yes | Product grid and filters |
| PDP | Yes | Product proof, copy, price, add to cart |
| Cart | Yes | Clear totals, shipping threshold, COD rules |
| Checkout | Yes | Razorpay/COD path |
| Order confirmation | Yes | Tracking expectations and WhatsApp support |
| Account/orders | Basic | Customer can see order status |
| Policy pages | Yes | Shipping, returns, privacy, terms |
| About | Yes | Brand trust and artist context |

## UX Requirements

- Mobile-first
- Fast PDP load
- Clear price and shipping threshold
- No account requirement before purchase
- WhatsApp-friendly product sharing
- COD minimum clearly enforced
- Prepaid discount surfaced without confusing checkout

## Catalog Scope

Recommended launch catalog:

- 30-50 SKUs
- Hero: card stickers, No Filter, original pins
- Support: selected earrings, magnets, stickers
- Filler/licensed IP hidden or deprioritized

## Completion Criteria

- Customer can browse products.
- Customer can add to cart.
- Customer can check out.
- Cart totals match Medusa totals.
- PDP and collection pages are analytics-ready.
- Mobile pages have no layout-blocking issues.

## Do Not Build Yet

- Artist portal
- Partner dashboard
- Custom product editor
- Full B2B portal
- Complex loyalty system
- Full personalization engine
